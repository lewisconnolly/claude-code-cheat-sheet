üß† Thinking Modes

Add these keywords to your prompt to increase reasoning depth:

  • think ~4,000 tokens ‚Äî Basic extended thinking
  • think hard / megathink ~10,000 tokens ‚Äî Enhanced reasoning
  • think harder / ultrathink ~32,000 tokens ‚Äî Maximum reasoning budget

üí° Press Alt + T to toggle thinking on/off. Use Ctrl+O to view thinking output.

üîß Custom Slash Commands

Create custom commands by adding .md files to .claude/commands/:

.claude/commands/review.md

---
description: Perform code review
allowed-tools: Read, Grep, Glob
---
Review this code for:
1. Security vulnerabilities
2. Performance issues
3. Best practices

.claude/commands/fix-issue.md (with arguments)

---
description: Fix GitHub issue
---
Fix issue #$ARGUMENTS following our
coding standards.

# Usage: /fix-issue 123

Use $ARGUMENTS placeholder to pass arguments. Commands in ~/.claude/commands/ are available globally.

ü™ù Hook Types

  • PreToolUse Before tool execution ‚Äî block, modify, or allow tools
  • PostToolUse After tool completion ‚Äî validate, format, log results
  • UserPromptSubmit When user submits prompt ‚Äî validate or enhance
  • PermissionRequest When permission dialog shown ‚Äî auto-allow or deny
  • Notification When Claude sends notifications
  • Stop When agent finishes responding
  • SubagentStop When a subagent finishes
  • SessionStart / SessionEnd Session lifecycle events
  • PreCompact Before compaction operations

üîå Add MCP Servers

Add MCP servers via CLI:

# Add a server
claude mcp add <name> -- <command>

# Example: Add GitHub server
claude mcp add github -- npx -y \
  @modelcontextprotocol/server-github

# List configured servers
claude mcp list

# Remove a server
claude mcp remove <name>

Or add to .mcp.json in project root to share with team.

üêç Python SDK Usage 

import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions, AssistantMessage, TextBlock

async def main():
    options = ClaudeAgentOptions(
        allowed_tools=["Read", "Write", "Bash"],
        permission_mode='acceptEdits',  # Auto-accept file edits
        cwd="/path/to/project"
    )
    
    async for message in query(
        prompt="Find and fix the bug in auth.py",
        options=options
    ):
        if isinstance(message, AssistantMessage):
            for block in message.content:
                if isinstance(block, TextBlock):
                    print(block.text)

asyncio.run(main())

Install: pip install claude-agent-sdk ‚Äî Requires Claude Code CLI: npm install -g @anthropic-ai/claude-code

üìã Log Tool Input

Add to .claude/settings.json to log all tool inputs to a file:

{
  "hooks": {
    "PostToolUse": [  // Or "PreToolUse" or "Stop", etc
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "jq . > post-log.json"
          }
        ]
      }
    ]
  }
}

Hook receives JSON via stdin with tool_name, tool_input, and tool_response (for PostToolUse). Exit code 2 blocks the action and shows error to Claude. You can use this information to build custom hook commands.
